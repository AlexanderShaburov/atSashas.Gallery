### Bug Fix: Journey return losing selected block
- **Symptom**: After selecting a block via Journey from Home Composer, returning showed no new block
- **Root cause**: `editSessionsDataStore.setSnapshot()` sets BOTH `snapshot` and `draft` to the same value. The bootstrap code called `saveDraft(newDoc)` first, then `setSnapshot(data)` â€” wiping the draft that contained the new block.
- **Fix**: Swapped call order â€” `setSnapshot(data)` first (establishes baseline), then `saveDraft(newDoc)` (sets draft with new block, preserving snapshot)
