# Report: Architecture Compliance Remediation — 2026-03-06

## Summary
Fixed 3 violations identified in the architecture compliance audit:
1. Unguarded `navigate()` in AdminHeader logout (bypassed Journey guards — ADR-001)
2. Unguarded `navigate()` in PublicStreamSession exit (bypassed Journey guards — ADR-001)
3. Event editor stored global data (`events[]`, `draft`) in React Context state instead of external stores (ADR-003)

## Work Item 1: `useGuardedNavigate` Hook

**Created** `features/admin/shared/hooks/useGuardedNavigate.ts`

- Wraps `useNavigate()` with `journeySessionStore.hasActiveSession()` check
- On active journey: shows `confirm()` dialog with same message as `GuardedNavLink`
- On confirmation: clears journey session, then navigates
- Returns a single `guardedNavigate(to: string)` callback

**Pattern**: Complements the existing `GuardedNavLink` (for `<NavLink>` elements) with an imperative equivalent for programmatic navigation — both use the same UX message and clear logic.

## Work Item 2: AdminHeader Logout Guard

**Modified** `features/admin/shared/ui/adminHeader/AdminHeader.tsx`

- Replaced `useNavigate()` import with `useGuardedNavigate`
- `handleLogout()` now calls `guardedNavigate('/admin/login')` instead of raw `navigate()`
- Removed unused `react-router-dom` import

**Before**: Clicking Logout during an active Journey silently abandoned the session.
**After**: User sees confirmation dialog; journey is properly cleared on confirmation.

## Work Item 3: PublicStreamSession Exit Guard

**Modified** `features/admin/publicStream/publicStreamSession/PublicStreamSession.context.tsx`

- Replaced `useNavigate()` with `useGuardedNavigate()`
- `exit` callback now calls `guardedNavigate('/admin')` instead of raw `navigate('/admin')`
- Removed unused `react-router-dom` import

**Before**: Clicking Exit in Home Composer during an active Journey (e.g. after dispatching to Block editor) silently abandoned the session.
**After**: User sees confirmation dialog; journey is properly cleared on confirmation.

## Work Item 4: Event Editor — External Store Migration

**Modified** `features/admin/eventEditor/eventEditorSession/EventEditorSession.context.tsx`

This was the largest change. The `EventEditorSession` interface remained **identical** — all changes were internal.

### 4a: Events list → `eventsStore` (external)

| Before | After |
|--------|-------|
| `const [events, setEvents] = useState<EventData[]>([])` | `const catalog = useStoreData(eventsStore)` |
| `loadEvents()` called `eventsAdminApi.getAll()` and wrote to local state | `loadEvents()` calls `refreshEvents()` which writes to `eventsStore` |
| After save/delete: `void refreshGlobalEvents()` (React Context dependency) | After save/delete: `await refreshEvents()` + `invalidateEventsCache()` (direct API) |

Events array is now derived: `useMemo(() => catalog ? Object.values(catalog.events) : [], [catalog])`

### 4b: Editor draft → `editSessionsDataStore` (external)

| Before | After |
|--------|-------|
| `const [draft, setDraft] = useState<EventDraft>(EMPTY_DRAFT)` | `useSessionDataStore<EventDraft>(editorKey)` with dynamic key |
| Draft mutations via `setDraft(prev => ...)` | Draft mutations via `editSessionsDataStore.saveDraft(key, next)` |

- Added `const [editorKeyId, setEditorKeyId] = useState<string | null>(null)` to track active editor key
- `EditorKey` is derived: `{ kind: 'events', id: editorKeyId }` (already supported in `editorKey.types.ts`)
- `selectEvent(id)` → writes snapshot via `editSessionsDataStore.setSnapshot()`, sets key
- `createNew()` → uses key `'__new__'`, writes `EMPTY_DRAFT` to store
- `setDraftField` / `onTitleChange` → reads imperatively from store, writes via `setStoreDraft()`
- `save()` / `deleteEvent()` → clears store session after success
- `back()` → clears store session, resets key

### 4c: Bootstrap reads stores imperatively

- Calls `refreshEvents()` (populates `eventsStore`)
- For journey `mode: 'edit'`: reads `eventsStore.getSnapshot()` imperatively, writes to `editSessionsDataStore`
- For journey `mode: 'select'`: just sets `screenMode('select')`

### 4d: Transient UI state stays in Context (permitted per ADR-003)

`screenMode`, `isLoading`, `isSaving` remain as `useState` — these are transient UI state, not global truth.

### 4e: Cleanup on unmount

- Tracks `editorKey` via `useRef`
- Clears `editSessionsDataStore` entry on provider unmount

### 4f: Removed React Context dependency

- Removed `useRefreshEvents()` import from `shared/EventsProvider/EventsProvider`
- Replaced with direct calls to `refreshEvents()` (from `eventsAdminApi.ts`) and `invalidateEventsCache()` (from `shared/EventsProvider/eventsApi.ts`)
