# Wave 1 Complete: Data Plane — External Stores as Single Source of Truth

**Date**: 2026-02-28
**Branch**: release-mvp
**Wave**: 1 of 4 (Architecture Compliance Roadmap)

---

## Objective

Establish external stores as the single source of truth for shared data. Dissolve the "god-context" pattern where six React contexts held global collections in `useState`.

This wave was prioritized first because every subsequent fix (control plane cleanup, layer hygiene, rendering unification) depends on clear data ownership.

## Known Gap

**PublicStreamSession** still uses local `useState` for `savedDoc` / `draft` instead of `editSessionsDataStore`. This was noted in the audit as item 5 ("Move PublicStreamSession saved/draft into editSessionsDataStore"). The editor reads from domain stores for display data but manages its own edit state locally. This is a candidate for Wave 2 (Control Plane cleanup) as it relates to session editing patterns rather than data ownership.
